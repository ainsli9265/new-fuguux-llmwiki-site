# Web Accessibility Initiative (Wikipedia)

An encyclopedia article on WAI, the W3C body that produces WCAG (already covered in depth via [[wikipedia-2026-web-content-accessibility-guidelines]]) and its sibling specifications. Wikipedia articles are continuously edited, so `year` above reflects the access date (2026-06-24), not a fixed publication year.

## Key points

- WAI launched in 1997 as a World Wide Web Consortium (W3C) initiative "to improve the accessibility of the World Wide Web for people with disabilities," and develops its guidelines through the same process as the rest of W3C.
- Produces five specification families, only one of which (WCAG) this wiki had previously covered: WCAG (content), **ATAG** (Authoring Tool Accessibility Guidelines — for content-creation-tool developers, v2.0 finalized September 2015), **UAAG** (User Agent Accessibility Guidelines — for browser/media-player developers, v2.0 published December 2015), **WAI-ARIA** (Accessible Rich Internet Applications — declaring dynamic content's roles/states to assistive technology, became a W3C standard March 2014), and **XAG** (XML Accessibility Guidelines — development stopped in 2002).
- Cites a statistic on the business cost of inaccessibility: 71% of website visitors with disabilities will leave a site that isn't accessible.
- Organized into working groups (e.g. ATAG WG, Accessibility Guidelines WG, Evaluation and Repair Tools WG, User Agent Accessibility Guidelines WG) and interest groups handling education/outreach.
