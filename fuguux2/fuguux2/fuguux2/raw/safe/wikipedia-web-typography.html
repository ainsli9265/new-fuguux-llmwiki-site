Web typography

Web typography addresses design challenges specific to web pages, distinguishing itself from print typography through dynamic presentation capabilities: "pages intended for display on the World Wide Web have additional technical challenges and – given its ability to change the presentation dynamically – additional opportunities."

CSS1 and Foundational Font Properties

The first CSS specification identified five core font properties: font-family, font-style, font-variant, font-weight, and font-size. All fonts were identified purely by name; designers lacked styling mechanisms for fonts unavailable on client systems.

Web-Safe Fonts

Web-safe fonts represent computer typefaces reasonably expected to be present across operating systems and devices. They increase "the likelihood that content displays in their chosen font" by relying on fallback mechanisms when specified fonts are unavailable.

Microsoft's Core Fonts for the Web (1996–2002)

Microsoft launched this initiative in 1996 and terminated it in 2002. It distributed Arial, Courier New, Times New Roman, Comic Sans, Impact, Georgia, Trebuchet, Webdings, and Verdana under an EULA allowing free distribution. These fonts became "de facto fonts of the Web." Linux distributions typically exclude them by default.

Fallback Fonts and Font Stacks

The CSS font-family property accepts a comma-separated list of typeface names. Browsers sequentially attempt each listed font and fall through to the next if the preferred one is unavailable, ultimately defaulting to the browser's own default fonts if none match. Example:

font-family: "Nimbus Sans L", Helvetica, Arial, sans-serif;

Generic Font Families

Five generic font families provide a design-intent fallback of last resort:

  • sans-serif — lacking decorative markings; considered easier for screen reading
  • serif — featuring decorative markings; traditionally associated with printed materials
  • monospace — all characters equally wide
  • cursive — resembling handwriting; difficult to read at small sizes
  • fantasy — containing decorative or symbolic characters while representing specified characters

CSS Fonts Working Draft 4 (as of February 2024)

W3C proposes system-level font specification through ui tags with limited current browser support: system-ui, ui-serif, ui-sans-serif, ui-monospace, ui-rounded, emoji, math, and fangsong (a Chinese typeface between serif and cursive styles).

Web Fonts

Historical Development

CSS2 (1998) introduced the @font-face construct for remote font downloading, though font licensing concerns existed. Internet Explorer 4 (1997) pioneered support through Embedded OpenType (EOT) fonts using Microsoft's proprietary WEFT tool.

In 2006, Håkon Wium Lie initiated efforts promoting standard font formats over EOT. Support for TrueType and OpenType subsequently appeared in Safari 3.1, Opera 10, Mozilla Firefox 3.5, and Internet Explorer 9.

The Web Open Font Format (WOFF) was submitted to W3C in 2010 by Mozilla Foundation, Opera Software, and Microsoft. Google Fonts launched in 2010 under open-source licenses, reaching 800+ families by 2016. "As of 2016 a majority of sites use webfonts," demonstrating technology maturation.

File Format Specifications

TrueDoc: Developed by Bitstream in 1994. Supported native embedding in Netscape Navigator 4 (1996) but lost support in Navigator 6 due to open-source licensing constraints. Lost to Microsoft's competing EOT format.

Embedded OpenType (EOT): Supported in Internet Explorer since version 4.0. Employed DRM techniques preventing unauthorized font copying. A simplified subset, CWT (Compatibility Web Type, formerly EOT-Lite), was formally standardized.

SVG Fonts: A W3C standard that allowed multicolor and animated fonts as SVG graphic subsets before deprecation in SVG 2.0. Most browsers except IE and Firefox supported it; Chrome deprecated standalone usage. Firefox supports SVG OpenType since version 26.

TrueType/OpenType (TTF/OTF): Industry-standard formats. Support: Firefox 3.5+, Opera 10+, Safari 3.1+, Chrome 4.0+. IE9+ requires fonts with installable embedding permissions.

Web Open Font Format (WOFF): Combines OpenType or TrueType with compression and metadata. Support: Firefox 3.6+, Chrome 5+, Opera Presto, IE9+ (from March 2011), Safari 5.1+.

Unicode Fonts

A "Unicode font" maps glyphs to Unicode Standard code points. The designation "has become redundant since the vast majority of modern computer fonts use Unicode mappings." TrueType fonts support a maximum of 65,535 glyphs, insufficient for all 159,801 Unicode 17.0 characters.

Windows includes Microsoft Sans Serif and Lucida Sans Unicode for wide Unicode repertoires. Linux offers GNU Unifont and GNU FreeFont. ChromeOS features Google's Noto fonts, designed to support all Unicode-encoded scripts.

Alternatives (Pre-Web-Font Era)

Before web fonts became practical, designers addressed the lack of typographic control through several strategies — most now obsolete or anti-pattern:

  • Font substitution: replacing unavailable fonts with similar web-safe alternatives. The simplest approach; still valid as a fallback stack strategy.
  • Image replacement: overlaying text with images rendered in the desired typeface. This achieves typographic control but prevents text selection, increases bandwidth usage, harms search engine optimization, and creates accessibility barriers for users relying on assistive technology. Web fonts have made this approach obsolete for body text and headings.
  • Flash (sIFR): rendered selectable vector text in the desired font but required a proprietary plugin. Now obsolete.
  • JavaScript (VML/SVG): replaced text with vector graphics via VML (Internet Explorer) or SVG (other browsers). Now superseded by web fonts.
