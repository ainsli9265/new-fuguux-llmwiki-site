Responsive web design

Responsive web design (RWD) is "an approach to web design that aims to make web pages render well on a variety of devices and window or screen sizes from minimum to maximum display size to ensure usability and satisfaction."

The core metaphor: "Content is like water" — content adapts fluidly to its container, regardless of the container's shape or size.

Core Techniques

RWD employs three primary techniques:

  1. Fluid proportion-based grids: page element sizing uses relative units like percentages rather than absolute units like pixels or points, so layout columns flex proportionally as the viewport changes.
  2. Flexible images: images are sized in relative units so they scale within their containing element and do not overflow the layout.
  3. CSS3 media queries: allow different style rules to apply based on device characteristics — particularly the width of the rendering surface (browser window width or physical display size).

Historical Development

  • 2001: Audi.com launched as the "first major site" featuring adaptive layout capabilities, developed by a Razorfish team including Jürgen Spangl and Jim Kalbach.
  • 2004: Cameron Adams created an early demonstration of resolution-dependent design.
  • 2008–2009: Related terminology emerged — "flexible," "liquid," "fluid," "elastic" — and CSS3 media queries neared implementation.
  • May 2010: Ethan Marcotte coined the term "responsive web design" in an A List Apart article, defining its three components: fluid grids, flexible images, and media queries.
  • 2011: Marcotte published the book Responsive Web Design.
  • 2012: RWD ranked #2 in web design trends by .net magazine.
  • 2013: Mashable designated this "the Year of Responsive Web Design."
  • 2015: Google announced "Mobilegeddon," boosting the search ranking of mobile-friendly sites when searching from a mobile device.

Business and UX Significance

Mobile device usage drove RWD adoption. Mobile users came to represent the majority of website visitors, making responsive design increasingly essential for accessibility, usability across devices, and search-ranking visibility.

Challenges

  • Non-fluid advertisements and embedded videos can break responsive layouts.
  • Complex multi-device layout patterns are hard to design and test.
  • Testing and validation must cover a wide range of device sizes and capabilities.

Alternative and Complementary Approaches

Mobile-first design: a related concept that predates RWD, recommended especially for basic mobile phones lacking JavaScript or media query support. The approach designs for the smallest, most constrained device first, then progressively enhances for larger/more capable devices.

RESS (responsive design with server-side components): an approach advocated by Luke Wroblewski that combines client-side responsive techniques with server-side device detection, aiming to provide a user experience better optimized for mobile devices than purely client-side RWD. Solutions include device-specific banner ad formats, different landing-page URLs per platform, and Ajax-based ad-variant delivery.

Testing and Validation

Approaches range from mobile site validators and mobile emulators to simultaneous multi-device testing tools such as Adobe Edge Inspect. Chrome, Firefox, and Safari all include built-in responsive design viewport-resizing tools in their developer-tools panels.

User Interface Plasticity

RWD is an instance of user interface plasticity — "the capacity of a user interface to withstand variations of both the system's physical characteristics and the environment while preserving usability." A plastic UI maintains its usability and satisfaction goals even when screen size, input method, or rendering context varies significantly.
