# Making Site Search Fast and Relevant

Pattern Group J of *The Design of Sites* — three patterns (J1-J3) covering on-site search, from the search box itself through to the results page.

## Key points

- [[search-action-module]] (J1) — already a concept page in this wiki from Chapter 2's worked example; this chapter gives it full six-part pattern documentation for the first time, adding the search-selector default/threshold guidance that the earlier worked example already captured. Updated the existing page rather than duplicating it.
- [[straightforward-search-forms]] (J2) — new page. Why Boolean search syntax and exact-match-only engines frustrate customers, and how to compensate (plain-language phrasing, dictionary/synonym handling, defaulting category searches to "all").
- [[organized-search-results]] (J3) — new page. How to make a results page usable: relevant per-domain summaries, grouped/organized hits, real hyperlinked titles, log-file-driven tuning for common terms and misspellings, and support for common search tasks (repeated search box, large pagination targets).
