# Web Accessibility Initiative (WAI)

The World Wide Web Consortium (W3C) body responsible for Web accessibility standards, launched in 1997 "to improve the accessibility of the World Wide Web for people with disabilities." WAI develops its guidelines through the same process as the rest of W3C, organized into working groups (e.g. the Accessibility Guidelines Working Group, which maintains [[wcag]]) and interest groups covering education and outreach.

WAI produces five specification families, each aimed at a different part of the Web's supply chain rather than content authors alone:

- **[[wcag|WCAG]]** (Web Content Accessibility Guidelines) — for anyone authoring Web content; see [[wcag]] for its full structure and history.
- **ATAG** (Authoring Tool Accessibility Guidelines, v2.0 finalized September 2015) — for developers of the tools content authors use (CMSes, page builders), covering both making the tool itself accessible and helping it produce accessible output.
- **UAAG** (User Agent Accessibility Guidelines, v2.0 published December 2015) — for developers of browsers and media players, the software that renders content for the end user.
- **[[wai-aria|WAI-ARIA]]** (Accessible Rich Internet Applications, a W3C standard since March 2014) — a technical vocabulary for marking up dynamic, JavaScript-driven content (custom widgets, live regions) so assistive technology can correctly interpret roles and state changes that plain HTML semantics don't cover; see [[wai-aria]] for its roles/states/properties model and the Five Rules of ARIA.
- **XAG** (XML Accessibility Guidelines) — guidance for accessibility in XML-based applications; development stopped in 2002 and it's effectively superseded by the other four.

**Business case:** WAI cites a statistic that 71% of website visitors with disabilities will leave a site that isn't accessible — a concrete number behind the more general accessibility argument already made in [[accessibility]].
